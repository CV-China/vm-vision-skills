## 1. 单相机旋转抓取场景

### 1.1 场景模型

#### 1.1.1 场景描述

单个上相机拍摄物料提取物料特征点实现定位引导，相机因视野限制会发生旋转及平移。

#### 1.1.2 机构限制

- 机构是XYR一体
- 相机运动随R轴转动

#### 1.1.3 相机运动类型

| 运动类型 | 说明 |
|---------|------|
| **相机旋转** | 相机随R轴旋转拍摄物料 |
| **相机平移** | 相机发生平移以覆盖更大视野 |
| **旋转+平移** | 相机同时发生旋转和平移 |

### 1.2 方案设计

#### 1.2.1 标定方案

**相机与机构做十二点标定**（平移旋转标定模块）

#### 1.2.2 偏差计算流程

| 步骤 | 操作内容 | 工具/方法 |
|-----|---------|----------|
| **1** | 相机旋转拍摄物料 | 提取稳定的特征点，并计算相机旋转角度（定位工具） |
| **2** | 将旋转拍照的图像特征点和角度转回基准图像坐标系 | 旋转拍照的图像特征点和角度转换回基准拍照姿态图像坐标系下（旋转公式） |
| **3** | 将特征点转换到机构坐标系 | 转换到机构坐标系（标定转换） |
| **4** | 偏差计算 | 实际偏差计算模型是一个常规的单点抓取模型（单点抓取） |

> **重要提示**：使用该方法需要先验证机构旋转是否会引起旋转中心偏移（验证旋转精度）。

### 1.3 方案验证

#### 1.3.1 验证方法

通过对比实际拍摄与推算结果来验证方案可行性：

| 验证项 | 说明 |
|-------|------|
| **相机0度拍照** | 基准位置拍摄，获取基准特征点 |
| **相机5度拍照** | 旋转5度拍摄，获取旋转后特征点 |
| **相机5度拍照推算** | 使用旋转公式推算5度位置特征点，与实际拍摄对比 |
| **相机10度拍照** | 旋转10度拍摄，获取旋转后特征点 |
| **相机10度拍照推算** | 使用旋转公式推算10度位置特征点，与实际拍摄对比 |

### 1.4 方案展示

#### 1.4.1 偏差计算方案

**步骤1：获取转换输入**

| 输入项 | 获取方式 |
|-------|---------|
| **像素旋转中心** | 从标定文件中获取 |
| **旋转角度** | 机构给出 / 通过视野中固定的目标计算 |

**步骤2：旋转图像坐标系转基准图像坐标系**

可通过变量计算或者脚本模块实现旋转公式。

**旋转公式**：
```
x' = (x - cx) × cos(θ) - (y - cy) × sin(θ) + cx
y' = (x - cx) × sin(θ) + (y - cy) × cos(θ) + cy
```

其中：
- (x, y)：旋转后的图像坐标
- (x', y')：转换回基准图像坐标系的坐标
- (cx, cy)：像素旋转中心
- θ：旋转角度（相机旋转角度取反）

**步骤3：偏差计算**

偏差计算模型为单点抓取模型，可直接使用单点抓取模块。

## 2. 旋转纠偏场景

### 2.1 方案设计

#### 2.1.1 标定方案

**相机与机构做十二点标定**（平移旋转标定模块）

#### 2.1.2 偏差计算流程

| 步骤 | 操作内容 | 工具/方法 |
|-----|---------|----------|
| **1** | 机构抓取物料旋转至下相机位拍摄物料 | 提取稳定的特征点，并获取相机旋转角度（定位工具） |
| **2** | 将旋转拍摄的图像特征点和角度转回基准图像坐标系 | 旋转拍照的图像特征点和角度转换回基准拍照姿态图像坐标系下（旋转公式） |
| **3** | 将特征点转换到机构坐标系 | 转换到机构坐标系（标定转换） |
| **4** | 偏差计算 | 实际偏差计算模型是一个常规的单点纠偏模型（单点纠偏） |

> **重要提示**：使用该方法需要先验证机构旋转是否会引起旋转中心偏移（验证旋转精度）。

### 2.2 拍照位变化情况

#### 2.2.1 拍照位平移

**场景描述**：相机拍照位发生平移变化。

**处理方法**：
- 平移量需在偏差计算中补偿
- 或使用十二点标定覆盖平移范围

#### 2.2.2 拍照位旋转

**场景描述**：相机拍照位发生旋转变化。

**处理方法**：
- 使用旋转公式将特征点转换回基准坐标系
- 再进行偏差计算

#### 2.2.3 拍照位平移+旋转

**场景描述**：相机拍照位同时发生平移和旋转变化。

**处理方法**：
- 先使用旋转公式转换坐标系
- 再补偿平移量
- 或使用十二点标定覆盖变化范围

## 3. 上下相机各自基准目标位旋转对位场景

### 3.1 方案设计

#### 3.1.1 标定方案

**上下相机分别与机构做十二点标定**（平移旋转标定模块）

#### 3.1.2 生产过程

| 步骤 | 操作内容 | 工具/方法 |
|-----|---------|----------|
| **1** | 下相机拍摄物料纠偏 | 下相机拍物料，提取需要对位的特征点，计算纠偏偏移量（单点纠偏） |
| **2** | 上相机旋转拍料盘 | 上相机旋转拍照，提取料盘图像特征点和角度转换回基准拍照姿态图像坐标系（旋转公式） |
| **3** | 上相机放置 | 计算上相机放置偏差（单点抓取） |
| **4** | 偏差计算 | 实际偏差为旋转放置偏差加修正后的纠偏偏差（目标位料盘发生旋转需要修正纠偏偏差） |

#### 3.1.3 关键要点

- 下相机纠偏计算常规单点纠偏
- 上相机目标位旋转时需使用旋转公式转换坐标
- 最终偏差需要叠加纠偏偏差和放置偏差
- 注意目标位旋转对纠偏偏差的影响

## 4. 上下相机旋转映射对位场景

### 4.1 方案设计

#### 4.1.1 标定方案

**步骤1：上下相机做映射标定**

- 映射标定搬运标定板过程中需要旋转θ角度并平移
- 正常取图做映射标定（映射标定）

**步骤2：下相机与机构做N点标定**

- 下相机旋转θ角度与机构做十二点标定（平移旋转标定模块）

#### 4.1.2 生产计算过程

| 步骤 | 操作内容 |
|-----|---------|
| **1** | 下相机旋转拍摄物料，提取需要对位的特征点及直线角度 |
| **2** | 上相机拍照，提取料盘图像特征点和对位直线起点终点 |
| **3** | 偏差计算 |

#### 4.1.3 偏差计算原理

**核心思想**：

实际偏差需将目标位坐标旋转θ角度，这样偏差计算模型就转换成平移映射对位模型。

**最终化简结果**：

```
最终偏差 = 单点映射对位偏差量 × 旋转矩阵
```

其中旋转角度为：**目标位拍照角度 - 对象位拍照角度**

#### 4.1.4 旋转矩阵

```
[cos(θ)  -sin(θ)  0]
[sin(θ)   cos(θ)  0]
[  0        0     1]
```

## 5. 总结与注意事项

### 5.1 场景对比

| 场景 | 标定方案 | 核心计算 |
|-----|---------|---------|
| **单相机旋转抓取** | 十二点标定 | 旋转公式 + 单点抓取 |
| **旋转纠偏** | 十二点标定 | 旋转公式 + 单点纠偏 |
| **上下相机各自基准旋转对位** | 上下相机分别十二点标定 | 旋转公式 + 单点纠偏 + 单点抓取 |
| **上下相机旋转映射对位** | 映射标定 + 下相机十二点标定 | 旋转公式 + 映射对位 |

### 5.2 关键公式

**旋转公式（图像坐标系）**：
```
x' = (x - cx) × cos(θ) - (y - cy) × sin(θ) + cx
y' = (x - cx) × sin(θ) + (y - cy) × cos(θ) + cy
```

**角度计算**：
```
θ_转换 = -θ_旋转  （相机旋转角度取反）
```

### 5.3 实施检查清单

#### 标定阶段

- [ ] 确认机构类型（XYR一体）
- [ ] 确认相机运动方式（旋转/平移/旋转+平移）
- [ ] 进行十二点标定（九点平移+三点旋转）
- [ ] 验证旋转中心是否偏移
- [ ] 保存标定文件

#### 验证阶段

- [ ] 在不同旋转角度拍摄物料
- [ ] 使用旋转公式推算特征点位置
- [ ] 对比推算结果与实际拍摄结果
- [ ] 验证旋转精度是否满足要求

#### 生产阶段

- [ ] 获取像素旋转中心（从标定文件）
- [ ] 获取旋转角度（机构给出或视觉计算）
- [ ] 使用旋转公式转换坐标系
- [ ] 使用标定转换模块转换到机构坐标系
- [ ] 使用单点抓取/纠偏模块计算偏差
- [ ] 验证定位精度

### 5.4 注意事项

| 注意点 | 说明 |
|-------|------|
| **旋转中心验证** | 使用前必须验证机构旋转是否会引起旋转中心偏移 |
| **角度方向** | 注意相机旋转角度与机构旋转角度的方向关系 |
| **坐标系方向** | 确认图像坐标系与机构坐标系的旋转方向一致性 |
| **旋转公式** | 旋转公式中的角度需要取反（相机旋转角度→坐标转换角度） |
| **精度验证** | 建议在多个旋转角度验证定位精度 |
