## 1. 场景概述

### 1.1 应用场景

单相机常规抓取场景适用于以下应用：
- 物料的抓取应用
- 算法需要返回给机构相对偏移坐标或绝对抓取坐标

### 1.2 机构限制

- 机构是XYR一体
- 相机安装在R轴上，可随XY一起平移，随R轴一起旋转
- 或相机静止安装

### 1.3 抓取流程导图

抓取流程分为三个阶段：
1. **标定** - 建立像素坐标与物理坐标的映射关系
2. **示教** - 建立基准像素坐标和物理坐标
3. **生产** - 计算偏差并输出抓取坐标

## 2. 标定

### 2.1 标定方案搭建

#### 2.1.1 图像采集

**相机配置**：
- 相机安装在机械臂上，随机械臂一起运动
- 或相机静止安装，拍摄固定位置的物料

#### 2.1.2 图像特征点提取

**特征提取工具选择**：
- 可使用高精度匹配模块
- 可使用圆查找、直线查找等模块提取特征点

**注意事项**：
- 需搭配位置修正模块修正ROI
- 确保特征点提取的稳定性和精度

#### 2.1.3 物理坐标获取

**获取方式**：
- 可使用接收数据和协议解析模块
- 解析出通信发送的机械手实时物理坐标

#### 2.1.4 平移旋转标定模块配置

**标定模块参数设置**：

| 参数 | 设置值 | 说明 |
|-----|--------|------|
| **标定类型** | 平移标定/平移旋转标定 | 根据吸嘴与旋转中心是否共轴选择；共轴选择平移标定，不共轴选择平移旋转标定 |
| **相机模式** | 相机运动/相机静止 | 根据实际场景选择 |
| **自由度** | 默认选项2 | 建议使用默认选项第二个自由度（缩放、旋转、纵横比、倾斜以及平移） |
| **平移次数** | 9次 | 默认9次 |
| **旋转次数** | 3次 | 十二点标定可设为3 |

#### 2.1.5 创建全局触发

**通信配置**：

可在通信管理添加通信设备，全局触发建立触发事件，实现PLC通信触发视觉流程。

**协议示例**：
- **标定时**：PLC发送 `(C1,x,y,r)` 给视觉，视觉匹配到C1则触发标定流程
- **生产时**：PLC发送 `(T1)` 给视觉，视觉触发抓取流程

> 详细协议需与PLC工程师协商确定。

---

### 2.2 标定流程

#### 2.2.1 标定的操作步骤（以相机运动为例）

**步骤1：准备物料**
- 在视野中放置一个物料或标定片

**步骤2：平移标定**
- 机构带着相机按Z字形等间距平移9次
- 平移中保持角度固定
- 每移动到位一次，需通讯发送物理坐标（如：`C1,5,5,0`）给视觉流程并触发标定流程拍照

**步骤3：旋转标定**
- 回到中心点（一般为标定第五点，第十一点和第五点重合）
- 机构带着相机顺/逆时针旋转3次
- 旋转过程中机械手XY坐标保持固定

**步骤4：完成标定**
- 整体标定轨迹呈Z字形平移+旋转

#### 2.2.2 标定结果查看

**查看项及判断标准**：

| 查看项 | 正常状态 | 异常处理 |
|-------|---------|---------|
| **标定轨迹** | 轨迹是否存在波动（建议等步长，方便通过轨迹排查特征提取是否正确） | 轨迹异常需重新标定 |
| **标定状态** | 1表示成功 | 0表示失败需重新标定 |
| **平移像素平均误差** | 根据实际精度要求判断 | 一般建议在一个像素以内 |
| **旋转像素平均误差** | 根据实际精度要求判断 | 一般建议在一个像素以内 |
| **角度旋转一致性** | 1表示旋转一致，-1表示旋转方向相反 | 需旋转机构查看旋转一致性是否判断准确 |

**标定文件生成**：
- 标定状态检测正常则生成标定文件
- 保存标定文件供后续示教和生产使用

## 3. 示教

### 3.1 示教目的

通过示教建立目标检测物的**基准像素坐标**和**物理坐标**之间的关系。

引导机械手以基准姿态抓住物料，并记录此时机械手坐标，然后回拍照位记录此时的图像特征点。

实际生产时以此时图像特征点作为基准，将生产时图像特征点和基准图像特征点转到物理坐标系计算偏差。

### 3.2 示教流程

**步骤1：搭建生产方案**
- 在软件上完成生产（抓取）方案搭建

**步骤2：准备物料**
- 放置生产用的物料在工作台上

**步骤3：调整抓取位置**
- 调整机构位置，使得机构的吸嘴或夹爪可以很好地对准物料
- 记录此时机构的物理坐标为**示教物理点**，填入单点抓取模块

**步骤4：定位到拍照位**
- 机构移动到拍照位置（即示教拍照物理点填入单点抓取模块，建议为标定时第五点）
- 拍摄物料，获得像素坐标

**步骤5：配置单点抓取模块**

1. 在软件上选中单点抓取模块，并双击打开参数配置界面
2. 填写**示教抓取物理点**（即示教时抓取物料的机械手物理坐标）
3. 手动填写或订阅**基准图像像素点**（即此时拍照位拍照获取的图像像素点）
4. 点击"执行"对单点抓取模块进行单次执行
5. 点击"**创建基准**"完成图像基准点的设置
6. 此时会弹窗提示"基准点创建成功！"

**步骤6：绑定像素点**
- 创建完成后，可将像素点绑定为运行图像点
- 若采用同一分支下来的特征点，则无需重新绑定

## 4. 生产

### 4.1 生产拍照位确认

**拍照位要求**：

| 要求项 | 说明 |
|-------|------|
| **位置** | 生产拍照位坐标为示教拍照位坐标（示教拍照物理点，要与标定第五点拍照位一致） |
| **偏移** | 可产生X/Y偏移（最终偏差需在单点抓取偏差的基础上加上拍照位变化量） |
| **姿态** | 拍照姿态（相机旋转角度）必须保持不变 |
| **工作距离** | 生产拍照位与标定时的工作距离必须保持不变 |

### 4.2 生产方案搭建

**方案要求**：
- 生产时需保证提取的特征点与示教图像特征点一致
- 确保特征提取模块的参数与示教时一致

### 4.3 单点抓取模块配置

**参数配置**：

| 参数 | 配置说明 |
|-----|---------|
| **输入方式** | 平移标定对应九点标定，平移旋转标定对应十二点标定。确定是按点输入还是按坐标输入，可以订阅模块也可以手动输入 |
| **像素点** | 在抓取场景中，需要输入两个像素点，分别是基准点和运行点。其中基准点的像素点的输入在示教过程中已经完成，这里生产时传入的为运行点，订阅关系可复用 |

### 4.4 触发流程与结果输出

**结果类型**：
- **相对坐标** - 相对于基准位置的偏移量
- **绝对坐标** - 机构抓取的目标位置

**数据发送流程**：

1. 使用格式化模块订阅单点抓取模块输出的偏差值/绝对坐标
2. 使用发送数据模块将结果发送给机械手
3. 机械手按照绝对坐标可实现引导定位

> 绝对坐标即为机构抓取的位置，机械手直接移动到该位置即可完成抓取。

## 附录：关键要点总结

### 标定阶段要点

- [ ] 确认标定类型（平移/平移旋转）
- [ ] 确认相机模式（运动/静止）
- [ ] 平移9次，旋转3次（十二点标定）
- [ ] 检查标定轨迹和误差
- [ ] 确认角度旋转一致性

### 示教阶段要点

- [ ] 放置物料在工作台上
- [ ] 调整机构位置使吸嘴对准物料
- [ ] 记录示教抓取物理点
- [ ] 移动到拍照位（建议标定第五点）
- [ ] 获取基准像素坐标
- [ ] 成功创建基准

### 生产阶段要点

- [ ] 拍照位与示教位一致
- [ ] 拍照姿态保持不变
- [ ] 工作距离保持不变
- [ ] 特征提取与示教一致
- [ ] 正确输出绝对坐标给机械手

## 抓取与纠偏场景对比

| 对比项 | 抓取场景 | 纠偏场景 |
|-------|---------|---------|
| **物料位置** | 物料在工作台上（固定） | 物料在机械手上（随机械手运动） |
| **示教动作** | 调整机构对准物料，记录抓取物理点，再回拍照位 | 以基准姿态吸取物料，直接记录物理坐标和拍照位像素 |
| **相机安装** | 相机可随机械手运动或静止 | 相机通常静止安装 |
| **应用场景** | 从固定位置抓取物料 | 纠正机械手上物料的位置偏差 |
