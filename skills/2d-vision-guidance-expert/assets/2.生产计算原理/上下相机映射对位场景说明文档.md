## 1. 场景概述

### 1.1 应用场景

上下相机映射对位场景适用于以下应用：
- 上相机拍目标（料盘），下相机拍对象（物料）的对位贴合场景
- 需要上下两个相机协同工作实现精确定位

### 1.2 机构限制

- 机构是XYR一体
- 有两个相机：目标相机（一般是上相机）和对象相机（一般是下相机）
- 目标相机安装在执行机构上或静止安装
- 对象相机倒装

### 1.3 对位流程导图

对位流程分为三个阶段：
1. **标定** - 建立像素坐标与物理坐标的映射关系
   - 下相机平移旋转标定
   - 上下相机映射标定
2. **示教** - 建立基准像素坐标和物理坐标
3. **生产** - 计算偏差并输出对位坐标

## 2. 标定

### 2.1 下相机平移旋转标定

#### 2.1.1 图像采集

**相机配置**：
- 下相机倒装，拍摄机械手上的物料

#### 2.1.2 图像特征点提取

**特征提取工具选择**：
- 高精度模板匹配：粗定位
- 边缘交点：精定位，获取图像点和角度信息

#### 2.1.3 物理坐标获取

**获取方式**：
- 可使用接收数据和协议解析模块
- 解析出通信发送的机械手实时物理坐标

#### 2.1.4 平移旋转标定模块配置

**标定模块参数设置**：

| 参数 | 设置值 | 说明 |
|-----|--------|------|
| **标定类型** | 平移标定/平移旋转标定 | 共轴选择平移标定；不共轴选择平移旋转标定 |
| **相机模式** | 相机运动/相机静止 | 根据实际场景选择 |
| **自由度** | 默认选项2 | 建议使用默认选项第二个自由度（缩放、旋转、纵横比、倾斜以及平移） |
| **平移次数** | 9次 | 默认9次 |
| **旋转次数** | 3次 | 十二点标定可设为3 |

#### 2.1.5 创建通信触发

**通信配置**：

可在通信管理添加通信设备，全局触发建立触发事件，实现PLC通信触发视觉流程。

**协议示例**：
- **标定时**：PLC发送 `(C1,x,y,r)` 给视觉，视觉匹配到C1则触发标定流程

> 详细协议需与PLC工程师协商确定。

#### 2.1.6 下相机平移旋转标定流程

**步骤1：准备物料**
- 机械手吸取标定片或物料移动至下相机视野内

**步骤2：平移标定**
- 机构带着相机按Z字形等间距平移9次，平移中保持角度固定
- 每移动到位一次，需通讯发送物理坐标（如：`C1,5,5,0`）给视觉流程并触发标定流程拍照

**步骤3：旋转标定**
- 回到中心点（一般为标定第五点，第十一点和第五点重合）
- 机构带着相机顺/逆时针旋转3次，旋转过程中机械手XY坐标保持固定

**步骤4：完成标定**
- 整体标定轨迹呈Z字形平移+旋转

#### 2.1.7 标定结果查看

**查看项及判断标准**：

| 查看项 | 正常状态 | 异常处理 |
|-------|---------|---------|
| **标定轨迹** | 轨迹是否存在波动（建议等步长，方便通过轨迹排查特征提取是否正确） | 轨迹异常需重新标定 |
| **标定状态** | 1表示成功 | 0表示失败需重新标定 |
| **平移像素平均误差** | 根据实际精度要求判断 | 一般建议在一个像素以内 |
| **旋转像素平均误差** | 根据实际精度要求判断 | 一般建议在一个像素以内 |
| **角度旋转一致性** | 1表示旋转一致，-1表示旋转方向相反 | 需旋转机构查看旋转一致性是否判断准确，若判断错误需调整参数重新标定 |

**标定文件生成**：
- 标定状态检测正常则生成标定文件
- 保存标定文件供后续示教和生产使用

### 2.2 上下相机映射标定

#### 2.2.1 映射标定步骤

**步骤1：准备标定物料**
- 放置标定物料于上相机视野内
- 物料要求：上下相机拍摄不同面时能同时拍摄到至少四个不共线的特征点

**步骤2：上相机拍照**
- 上相机在拍照位拍照

**步骤3：记录物理坐标**
- 移动机械手抓取物料，记录此时的物理坐标（为示教物理点）

**步骤4：下相机拍照**
- 抓取物料平移到下相机视野内，下相机拍照

**步骤5：搭建上下相机映射标定流程**
- 左右分别为上下相机采集的一一对应的图像特征点
- 用相机映射模块得到两个相机的映射矩阵

**步骤6：生成映射标定文件**
- 保存映射标定文件供后续生产使用

## 3. 示教

### 3.1 示教目的

通过示教建立目标检测物的基准像素坐标和物理坐标之间的关系。

记录映射标定时吸取物料时的物理坐标（示教物理点），填入单点映射对位模块。

### 3.2 示教流程

**步骤1：记录示教物理点**
- 在映射标定时，记录机械手抓取物料时的物理坐标
- 将该坐标填入单点映射对位模块的示教物理点参数

## 4. 生产

### 4.1 下相机拍照流程

#### 4.1.1 流程搭建

搭建下相机生产流程，提取物料特征点。

#### 4.1.2 数据传递

将下相机提取的对象特征点和直线角度传给全局变量，以便上相机位使用。

---

### 4.2 上相机拍照流程

#### 4.2.1 上相机拍照位确认

**拍照位要求**：

| 要求项 | 说明 |
|-------|------|
| **位置** | 上相机拍照位坐标与标定拍照位保持一致 |
| **姿态** | 拍照姿态必须保持不变 |
| **工作距离** | 上相机拍照位与标定时的工作距离必须保持不变 |

#### 4.2.2 上相机拍照方案搭建

**方案要求**：
- 生产时需保证下相机提取的对象特征点与上相机提取的目标图像特征点一致
- 直线查找方向一致

#### 4.2.3 单点映射对位模块配置

**参数配置**：

| 参数 | 配置说明 |
|-----|---------|
| **对象像素点** | 订阅下相机传送到全局变量的特征点 |
| **目标像素点** | 订阅上相机提取的特征点 |
| **目标直线起点终点** | 订阅上相机提取的直线起点终点 |
| **标定文件** | 加载上下相机标定的标定文件 |

---

### 4.3 实际生产流程

**步骤1：下相机拍照**
- 机械手抓取物料移动到下相机拍照位
- 触发下相机拍照流程

**步骤2：上相机拍照**
- 机械手移动到上相机拍照位
- 触发上相机拍照流程

**步骤3：计算对位坐标**
- 单点映射对位模块结果分为相对坐标及绝对坐标
- 绝对坐标则为机构抓取的位置

**步骤4：发送结果**
- 使用格式化模块订阅单点映射对位模块输出的坐标值
- 使用发送数据模块将结果发送给机械手
- 机械手按照绝对坐标可实现引导定位
