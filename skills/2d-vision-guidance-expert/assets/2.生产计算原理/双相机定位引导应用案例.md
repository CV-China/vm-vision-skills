## 1. 场景介绍

### 1.1 应用场景

**场景描述**：大物料定位引导场景，视觉根据物料姿态发送抓取坐标给机械手，引导机械手抓取大物料（相机架设高度有限制）。

### 1.2 物料信息

| 参数 | 数值 |
|-----|------|
| 长度 | 80cm |
| 宽度 | 60cm |
| 高度 | 10mm |

### 1.3 场景解析

可采用**双相机拍摄大物料对角并求中点**的方式实现单点抓取。

### 1.4 项目难点

1. **坐标系统一**：如何将两个相机的坐标系统一到同一坐标系下
2. **偏差计算**：如何计算抓取偏差

## 2. 方案设计

### 2.1 标定方案

#### 2.1.1 标定板标定（两个静止相机）

**标定目的**：两个相机分别进行标定板标定，将两个图像坐标系统一到标定板坐标系。

**实施步骤**：
1. 放置标定板于两个相机视野内
2. 保证两个相机都能拍摄到部分标定板
3. 分别进行标定板标定

#### 2.1.2 N点标定（主相机与机构）

**标定目的**：获取主相机图像坐标系与机构物理坐标系的转换矩阵，便于将目标转换到物理坐标系求其物理偏移量。

**标定物料选择**：
- 选择标准件进行标定
- 保证物料特征点能稳定提取
- 提取精度高

**具体操作方法**：
- 本场景属于相机静止场景
- 采用机构携带标准件来移动旋转的方式采集成对的图像点和物理点
- 或采用机构吸放物料的方式来进行图像特征点与机构物理点的采集
- 标定过程走九组平移点位，然后回到第五点进行三次旋转
- 最终生成标定矩阵

### 2.2 偏差计算方案

#### 2.2.1 坐标转换流程

**步骤1：主辅相机像素点转换到标定板坐标系**

```
CalibBoard_主 = M1_主 × Image_主
CalibBoard_辅 = M1_辅 × Image_辅
```

**步骤2：主辅相机映射到标定板坐标系联立求中点**

```
CalibBoardCent = (CalibBoard_主 + CalibBoard_辅) / 2
```

**步骤3：中点坐标转换到主相机图像坐标系**

```
ImageCent_主 = (M1_主)的逆 × CalibBoardCent
```

**步骤4：将中点坐标转换到机构物理坐标系求偏差**

```
WorldCent = M2_主 × ImageCent_主
```

#### 2.2.2 偏差计算步骤

| 步骤 | 操作内容 | 工具/方法 |
|-----|---------|----------|
| **1** | 主辅相机分别拍摄面板对角 | 提取对角稳定的特征点（定位工具） |
| **2** | 将两特征点统一到一个坐标系 | 转换到标定板坐标系并求中点坐标（标定转化，点点测量） |
| **3** | 中点坐标转换机构坐标系 | 中点坐标转换到主相机坐标系下，再用主相机和N点标定的标定矩阵转换到机构物理坐标系（标定转化） |
| **4** | 偏差计算 | 实际偏差计算模型是一个常规的单点抓取模型（单点抓取） |

## 3. 方案展示

### 3.1 标定方案实施

#### 3.1.1 上下相机分别进行标定板标定

**标定板标定目的**：两个相机分别做标定板标定，将两个图像坐标系映射到同一个标定板坐标系下。

**标定板选择**：

| 标定板类型 | 说明 |
|-----------|------|
| 棋盘格标定板 | 标准棋盘格图案 |
| 圆标定板 | 圆点阵列图案 |
| 海康Ⅰ/Ⅱ型标定板 | 可以使用VM提供的海康标定板工具生成对应大小的标定板，然后打印获取 |

> **注意**：生成标定板的大小应能同时覆盖两个相机视野。

**具体操作方法**：
1. 放置标定板于两个相机视野内，保证两个相机都能拍摄到部分标定板
2. 海康Ⅰ/Ⅱ型标定板每个棋盘格顶点的坐标都有记录，所以直接进行标定板标定即可
3. 其它类型标定板需要进行坐标偏移，将两个相机的在标定板坐标系上的坐标零点移动到同一个位置

#### 3.1.2 主相机与机构做N点标定

**N点标定目的**：获取主相机图像坐标系与机构物理坐标系的转换矩阵，便于将目标转换到物理坐标系求其物理偏移量。

**标定物料选择**：
- 选择标准件进行标定
- 保证物料特征点能稳定提取
- 提取精度高

**具体操作方法**：
1. 本场景属于相机静止场景
2. 采用机构携带标准件来移动旋转的方式采集成对的图像点和物理点
3. 或采用机构吸放物料的方式来进行图像特征点与机构物理点的采集
4. 标定过程走九组平移点位，然后回到第五点进行三次旋转
5. 最终生成标定矩阵

### 3.2 偏差计算方案实施

#### 3.2.1 单点抓取方案

**第一步：提取面板物料两个对角角点**
- 采用模板匹配进行粗定位
- 使用边缘交点进行精定位

**第二步：统一坐标系**
- 将两个相机提取的图像角点经过标定转换映射到同一个标定板坐标系

**第三步：获取点位和角度**
- 使用点点测量模块计算两点的中点及连线的角度

**第四步：保存基准位**
- 移动物料至基准位置
- 提取中点坐标、基准角度及示教物理坐标
- 保存单点抓取模块

**第五步：计算偏差并输出**
- 移动物料运行流程
- 单点抓取模块输出的物理值就是此时的抓取坐标
- 通过通信发送给机械手引导机械手实现抓取

#### 3.2.2 单点纠偏方案

**纠偏流程**：
1. 运行流程，计算偏差并输出
2. 单点纠偏模块输出的物理值就是此时的纠偏坐标
3. 通过通信发送给机械手引导机构移动

## 4. 知识点总结

### 4.1 方案制定

针对大物料场景：
- 可以选择**两个相机拍摄对角求中点**实现单点抓取
- 也可以增加相机数量来降低扰动，例如可以采用**四个相机拍摄四个角**，最后求中点实现单点抓取/纠偏
- VM常规定位的思路是**统一坐标系**，然后点对点计算偏差（需要找到稳定的特征点）

### 4.2 标定板标定

- 会使用海康标定板生成工具
- 其它标定板需要注意进行相机的坐标偏移，将不同图像坐标系在标定板坐标系上的坐标零点移动到同一个位置

### 4.3 坐标系统一

- 将不同的图像坐标系、标定板坐标系都统一到机构坐标系中计算偏差
- 需要注意标定转换模块的坐标类型选择

### 4.4 偏差计算

- 可以使用运算工具箱中的**单点抓取**、**纠偏**以及**单点映射对位**模块计算偏差
- 也可以自己使用脚本计算偏差
- **共轴场景**：统一完坐标系可以直接做差
- **不共轴场景**：可以引入旋转公式计算偏差，平移旋转标定的旋转中心就是标定转换后的原点

### 4.5 数据发送

- 通过**格式化模块**将对应的数据先进行收集
- 使用**发送数据模块**将坐标发送给机械手引导机构移动
