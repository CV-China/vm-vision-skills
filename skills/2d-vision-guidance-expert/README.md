# 2D 机器视觉定位引导专家 Skill 使用指南

> 适用平台：Claude Code
> 适用领域：基于 VisionMaster 平台的 2D 机器视觉定位引导

## 一、概述

`2d-vision-guidance-expert` 是一个资深 2D 机器视觉定位引导专家技能，精通标定原理、生产计算和误差分析，基于 VisionMaster 平台为工业现场工程师提供可落地的定位引导解决方案。

**核心能力：双模式工作**

| 模式 | 入口 | 产出 |
|------|------|------|
| **方案设计** | 描述应用场景（"怎么做""用什么方案"） | 标定 + 生产计算完整方案 HTML |
| **问题诊断** | 描述问题现象（"为什么不准""精度差"） | 根因分析 + 分级排查清单 HTML |

**知识库规模**：13 篇专业文档，覆盖标定原理（3 篇）、生产计算原理（7 篇）、定位误差分析（3 篇），另含 14 个 VM 标准流程图、6 个已验证输出样本。

技能保证以下事项：

- 每次回答引用至少一篇知识库文档，确保建议有据可查
- 方案设计前强制采集 6 项关键信息（相机数量、安装方式、应用类型、旋转需求、共轴/偏心、标定条件），信息不全不输出方案
- 所有正式输出为独立 HTML 文件（内嵌 CSS，浏览器直接打开即可查看彩色卡片和可视化流程图）
- VM 流程图通过 `vm_flow_builder.py` 预制函数生成，禁止手写 SVG 坐标
- 不输出代码（Python/C++/C#），不替代 VM 软件操作手册，不处理 3D 视觉或深度学习问题

## 二、标准使用流程

### 方案设计模式

#### Step 1：描述场景

用自然语言说明你的应用场景，例如：

> "我有一个单相机固定在机台上，需要引导机械手去抓取料盘上的工件。机械手是 XYR 一体式，吸嘴不在旋转中心。"

#### Step 2：回答 6 项关键信息确认

技能会逐一确认以下信息（已描述清楚的会直接确认，缺失的会合并提问）：

1. **相机数量与安装方式**（静止/运动）
2. **应用类型**（抓取/纠偏/对位）
3. **机构类型**（XYR 一体/分离轴）
4. **有无旋转需求**（是否需要输出角度偏差）
5. **工具是否共轴**（工具中心是否等于旋转中心）
6. **标定条件**（有无标定板、标定板大小）

#### Step 3：技能沿决策树定位场景

6 项信息确认后，技能沿 5 级决策树自动定位到具体场景（覆盖 16 种叶节点场景）：

- 级别 1：相机数量 → 级别 2：安装方式/标定条件 → 级别 3：应用类型 → 级别 4：旋转/对位方式 → 级别 5：共轴/偏心

#### Step 4：获得完整方案 HTML

技能输出独立 HTML 文件，包含 7 张彩色卡片：

1. **场景分析** — 一句话描述 + 关键参数确认表
2. **定位方案思路** — 分步思路图（设计者思考路径）
3. **VM 视觉流程图** — SVG 流程图（标定流程 + 生产流程）
4. **操作要点** — 标定（蓝）/ 示教（绿）/ 生产（橙）三阶段编号步骤
5. **注意事项** — 最多 3 条警告
6. **参考文档** — 相关文档列表

#### Step 5：方案自动导出

方案 HTML 自动保存到工作目录 `export/` 文件夹，文件名格式：`YYYYMMDD-HHmm_方案_关键词.html`。

> 本次方案已导出到 `export/方案_xxx.html`，用浏览器打开即可查看彩色卡片方案。

---

### 问题诊断模式

#### Step 1：描述问题现象

用自然语言描述你遇到的问题，例如：

> "十二点标定后，角度偏差越大抓取越不准。旋转 5 度时偏差 0.3mm，旋转 30 度时偏差 2mm。"

#### Step 2：补充误差数据

技能会追问以下关键信息：

- 问题现象（偏差大/重复性差/某些位置不准/系统性偏差）
- 误差数据（具体数值）
- 系统配置（相机/机构/标定方式）
- 已尝试的解决方法

#### Step 3：5Why 根因分析

技能逐层追问：表面现象 → 直接原因 → 根本原因 → 涉及子系统 → 相互影响。

#### Step 4：获得分级排查方案 HTML

技能按 **机构及成像 → 标定过程 → 示教过程 → 生产过程** 四步排查流程，输出三级清单：

- **立即检查（5 分钟）** — 快速排除项
- **短期排查（当天）** — 详细诊断项
- **深度排查（需规划）** — 系统性优化项

#### Step 5：（可选）提供标定文件进行深度分析

在标定排查阶段，技能会主动询问是否有标定文件（`.xml`）。如果提供，`calib_analyzer.py` 自动解析并输出 12 项检测报告（含 pass/fail 判定、标准值、排查方向），嵌入诊断报告中。

## 三、典型场景速查

### 三大底层运算模型

所有定位引导场景最终归约到三种 VM 运算模型：

| 场景类型 | VM 核心模块 | 适用条件 |
|---------|-----------|---------|
| **单点抓取** | `单点抓取` | 物料在工作台，引导机构去目标点 |
| **单点纠偏** | `单点纠偏` | 物料在机构上，引导机构使物料回基准位 |
| **单点映射对位** | `单点映射对位` | 上下相机贴合，目标特征和对象特征均偏移 |

### 场景选择速查

| 你的场景 | 推荐标定 | 推荐计算模型 |
|---------|---------|-------------|
| 单相机抓取，无旋转，共轴 | 九点标定 | 单点抓取 |
| 单相机抓取，无旋转，偏心 | 十二点标定 | 单点抓取 |
| 单相机抓取/纠偏，有旋转 | 十二点标定 | 旋转抓取/纠偏 |
| 单相机纠偏，无旋转 | 九点/十二点标定 | 单点纠偏 |
| 上下相机贴合对位 | 映射标定 + N 点 | 单点映射对位 |
| 双相机同侧拍对角（大物料） | 映射标定 + 平移旋转标定 | 单点抓取/纠偏 |
| 分离轴机构 | N 点标定 + 符号修正 | 按场景选择 |

### 十二点标定触发条件

满足以下**任意一条**即需十二点标定（含旋转中心标定）：

- **有旋转需求**（需要输出角度偏差，R 轴旋转补偿）
- **工具偏心**（吸嘴/夹爪中心与 R 轴旋转中心不重合）

只有无旋转需求 **且** 工具共轴，才使用九点标定。

### 常见问题快速索引

| 问题现象 | 可能原因 | 首要检查 |
|---------|---------|---------|
| 重投影误差大 | 标定点提取不准 | 图像质量和坐标对应 |
| 角度越大偏差越大 | 旋转中心标定不准 | 旋转中心拟合精度 |
| 系统性偏差 | 坐标系方向错误 | 图像与机械手坐标系对应 |
| 重复性差 | 机构精度/光源不稳定 | 静态测试 + 动态测试 |
| 某些位置不准 | 标定未覆盖/畸变 | 标定点分布和畸变校正 |
| 纠偏后仍不准 | 基准位置设置错误 | 重新设定基准位置 |
| 上下对位失败 | 平台运动方向错误 | 验证平台运动方向 |

## 五、触发方式

在对话中直接描述你的需求即可，技能会根据关键词自动触发。**触发词包括**：

标定、抓取、纠偏、对位、定位、精度、偏差、重复性、旋转中心、分离轴、九点标定、十二点标定、映射标定、VM 定位

也可以显式调用：

```
/2d-vision-guidance-expert
```

## 六、常见问题

| 现象 | 原因 / 处理 |
|------|------------|
| 技能没有触发 | 检查是否使用了触发词；可显式调用 `/2d-vision-guidance-expert` |
| 方案输出前被追问很多问题 | 这是技能强制信息采集流程，6 项关键信息不全不会输出方案，确保方案针对你的场景 |
| HTML 流程图节点错位 | 可能临时脚本手写了 SVG 坐标（违反硬约束）；反馈给技能维护者检查 `vm_flow_builder.py` 调用方式 |
| 标定文件分析报错 | 确认 XML 文件是 VisionMaster 平移旋转标定/N 点标定模块生成的 calib.xml |
| 输出中引用的文档找不到 | 知识库文档位于 skill 目录内，链接使用相对路径；确认 skill 目录结构完整 |
| 想要某个场景的样本参考 | 查看 `samples/` 目录，已有 6 个覆盖单相机抓取、纠偏、双相机、上下对位等场景的 HTML 样本 |

## 七、不支持范围

- **3D 视觉**（点云、立体视觉、深度相机等）
- **深度学习**（目标检测、分类、分割模型选型与调优）
- **代码输出**（Python/C++/C# 等编程语言代码）
- **VM 软件操作手册**（仅提供关键操作提示，不替代官方文档）
- **非 VisionMaster 平台**（其他品牌视觉软件的标定与计算）

遇到上述请求，技能会直接告知超出范围。

## 八、输出归档说明

每次按模板完成正式输出后，技能会自动将结果保存到工作目录的 `export/` 文件夹：

- **文件格式**：独立 HTML（内嵌 CSS，浏览器可直接查看）
- **文件命名**：`YYYYMMDD-HHmm_模式_关键词.html`（如 `20260527-1430_方案_单相机抓取-偏心旋转.html`）
- **不导出**：信息采集中间过程、闲聊、不完整输出
- **优质样本**：经用户确认正确的方案，会同步到 `samples/` 供后续参考
