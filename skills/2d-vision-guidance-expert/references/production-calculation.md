### 标定阶段要点

**标定方案搭建**：
- 图像采集：相机安装在机械臂上随动，或静止拍摄固定位置物料
- 特征提取：高精度匹配（粗定位）→ 圆查找/直线查找+位置修正（精定位）→ 抓边输出
- 物理坐标获取：接收数据和协议解析模块，解析通信发送的机械手实时物理坐标

**平移旋转标定模块配置**：

| 参数 | 选项 | 说明 |
|------|------|------|
| 标定类型 | 共轴→平移标定（九点）；不共轴→平移旋转标定（十二点） | 不确定时选十二点 |
| 相机模式 | 相机运动/相机静止 | 根据安装方式选择 |
| 自由度 | 默认选项2 | 缩放+旋转+纵横比+倾斜+平移 |
| 平移次数 | 9次 | |
| 旋转次数 | 3次 | |

**标定操作（以相机运动为例）**：
1. 在视野中放置物料或标定片
2. 机构带着相机按Z字形等间距平移9次，平移中保持角度固定
3. 回到中心点（第五点），机构带着相机顺/逆时针旋转3次
4. 查看标定结果：状态=1、平移精度<1像素、旋转精度<0.5像素

### 示教阶段要点

1. 放置物料在工作台上，调整机构使吸嘴对准物料
2. 记录此时机构物理坐标为示教物理点
3. 机构移动到拍照位（记录示教拍照位，标定第五点），拍摄物料获得基准像素坐标
4. 配置单点抓取模块：填写示教物理点、基准像素点，执行后"创建基准"

### 生产阶段要点

**拍照位要求**：位置与示教拍照位一致（可产生X/Y偏移，但需要在单点抓取后添加**变量计算**模块补偿X/Y偏移量）；拍照姿态必须不变；工作距离不变

**单点抓取模块输出**：
- 相对坐标：相对于基准位置的偏移量
- 绝对坐标：机构抓取的目标位置（直接移动到该位置）

### 抓取 vs 纠偏关键区别

| 对比项 | 抓取 | 纠偏 |
|--------|------|------|
| 物料位置 | 在工作台上（固定） | 在机械手上（随机械手运动） |
| 示教动作 | 调整机构对准物料→记录→回拍照位 | 基准姿态吸取→直接记录物理坐标和像素 |
| 输出公式 | 基准物理点 + 物理偏差 | 基准物理点 - 物理偏差 |
| 相机安装 | 可随机械手运动或静止 | 通常静止安装 |

> ⚠️ **纠偏示教关键约束**：创建基准时，机构必须移动到**标定第五点（旋转标定坐标）**处拍照，不能在任意位置拍照创建基准。标定第五点是标定坐标系的原点，只有在此处拍照，基准像素坐标才与标定矩阵对应，偏差计算才正确。

### VM方案流程图

```
├─────────────────────────────────────────────────────────────┤
│                        示教阶段                              │
│  记录映射标定时吸取物料的物理坐标 → 填入单点映射对位模块       │
├─────────────────────────────────────────────────────────────┤
│                        生产阶段                              │
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐     │

│  │(粗→精→抓边)  │   │(粗→精→抓边)  │   │ 输出对位偏差  │     │
│  └──────────────┘   └──────────────┘   └──────────────┘     │
└─────────────────────────────────────────────────────────────┘
```

### 映射标定操作

> ⚠️ **上下相机移动映射的核心约束**：上下相机视野不重叠，标定板需从上相机位**只平移不旋转**移动到下相机位分别拍照。这种映射关系依赖生产时上相机和下相机拍照位与映射标定时完全一致才能成立——生产过程中两个相机的拍照位必须固定，不能改变。

1. 将标定板放置于上相机视野内，上相机在拍照位拍照，提取特征点
2. 移动机械手抓取标定板，记录物理坐标（= 示教物理点）
3. 将标定板**只平移**（不旋转）到下相机视野内，下相机拍照，提取对应特征点
4. 搭建映射标定流程：左右分别为上下相机采集的一一对应的图像特征点（至少4组不共线）
5. 用相机映射模块得到两个相机的映射矩阵
6. 生产时上相机和下相机的拍照位必须与映射标定时完全一致，否则映射关系失效

### 单点映射对位模块配置

| 参数 | 说明 |
|------|------|
| 对象像素点 | 订阅下相机传送到全局变量的特征点 |
| 目标像素点 | 订阅上相机提取的特征点 |
| 目标直线起点终点 | 订阅上相机提取的直线 |
| 标定文件 | 加载上下相机标定的标定文件 |

## 4. 4种拍照位旋转定位场景

### 场景1：单相机拍照位旋转抓取

```
┌──────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ 相机旋转  │──→│ 提取特征点    │──→│ 旋转公式转换   │──→│ 单点抓取模块   │
│ 拍摄物料  │   │ +旋转角度    │   │ (转回基准坐标) │   │ 输出抓取坐标  │
└──────────┘   └──────────────┘   └──────────────┘   └──────────────┘
```

**旋转公式**：
```
x' = (x - cx) * cos(theta) - (y - cy) * sin(theta) + cx
y' = (x - cx) * sin(theta) + (y - cy) * cos(theta) + cy
theta = -(相机旋转角度)，(cx, cy)为像素旋转中心（从标定文件获取）
```

### 场景2：拍照位旋转纠偏

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ 机构抓取物料  │──→│ 旋转至下相机  │──→│ 拍照+特征提取  │
│              │   │ 拍照位       │   │(粗→精→抓边)   │
└──────────────┘   └──────────────┘   └──────────────┘
                        │
                        ▼
              ┌──────────────┐   ┌──────────────┐
              │ 旋转公式转换  │──→│ 单点纠偏模块   │
              │ (转回基准)   │   │ 输出纠偏坐标   │
              └──────────────┘   └──────────────┘
```

### 拍照位旋转场景速查

| 场景 | 标定方案 | 核心计算 | VM关键操作 |
|------|---------|---------|-----------|
| 单相机旋转抓取 | 十二点标定 | 旋转公式 + 单点抓取 | 先旋转公式转换，再单点抓取 |
| 旋转纠偏 | 十二点标定 | 旋转公式 + 单点纠偏 | 先旋转公式转换，再单点纠偏 |
| 各自基准旋转对位 | 上下分别十二点 | 单点纠偏 + 旋转公式 + 单点抓取 | 纠偏偏差需修正后叠加 |

## 6. 多相机定位的坐标转换流程与归约准则

**核心准则**：所有多相机场景最终归约到单相机的三大常规场景（单点抓取/单点纠偏/单点映射对位）。

**标定架构**（适用于所有多相机场景）：
- 辅助相机 → 通过 **VM 映射标定模块** → 统一到主相机坐标系
- 主相机 → 通过 **平移旋转标定** → 机构物理坐标系
- 传入单点抓取/单点纠偏/单点映射对位的特征点，必须是**主相机坐标系下的像素坐标**

**场景**：大物料（如80cm×60cm），双相机拍对角求中点

```
┌────────────────┐   ┌────────────────┐
│ 主相机像素点    │   │ 辅相机像素点    │
└───────┬────────┘   └───────┬────────┘
        │                    │
        │            ┌───────▼────────┐
        │            │   标定转换      │  辅相机→主相机坐标系
        │            │ (VM映射标定矩阵)│
        │            └───────┬────────┘
        │                    │
        ▼                    ▼
┌────────────────────────────────┐
│ 统一到主相机坐标系               │
│ 联立求中点 (主+辅)/2 和角度      │
└───────────────┬────────────────┘
                ▼
       ┌──────────────────┐
       │ 输入单点抓取模块   │  主相机坐标系下的像素坐标
       │ (模块内部转物理)   │
       └──────────────────┘
```

**核心思路**：辅相机特征点先映射到主相机坐标系 → 联立计算 → 输入标准 VM 模块

## 7. 拍照位变化时的处理

### 常规要求
- 位置：与示教拍照位一致（标定第五点）
- 姿态：拍照姿态必须不变
- 工作距离：必须不变

### 拍照位平移

```
最终偏差 = 单点抓取/纠偏偏差 + 拍照位变化量
```

- 平移偏差做差无需修改
- 旋转偏差构建的是向量，平移不变
- 示教位需加上平移量才是真实示教点
- 也可用十二点标定覆盖平移范围

### 拍照位旋转

```
使用旋转公式将特征点转换回基准坐标系
theta_转换 = -theta_旋转
```

### 拍照位平移+旋转

```
先旋转公式转换坐标系 → 再补偿平移量
```

### 关键原则
- 旋转公式使用的像素旋转中心从标定文件中获取
- 使用前必须验证机构旋转是否会引起旋转中心偏移
- 注意图像坐标系与机构坐标系的旋转方向一致性

## 附录：关键公式汇总

**旋转公式（图像坐标系）**：
```
x' = (x - cx) * cos(theta) - (y - cy) * sin(theta) + cx
y' = (x - cx) * sin(theta) + (y - cy) * cos(theta) + cy
theta = -(相机旋转角度)
```

**共轴/不共轴与十二点标定的关系**：
- 需要输出角度偏差（有旋转需求）→ 十二点标定
- 工具偏心（吸嘴与旋转中心不共轴）→ 十二点标定
- 无旋转需求 **且** 工具共轴 → 九点标定

**坐标系统一原则**：所有定位引导的核心思路是"统一坐标系"，将不同图像坐标系通过标定矩阵统一到机构物理坐标系，再进行点对点偏差计算。

**抄平原则**：标定、示教和生产三个阶段的拍照位、拍照姿态和工作距离必须保持一致。如无法一致，需在偏差计算中补偿拍照位变化量（平移）或使用旋转公式（旋转）。
