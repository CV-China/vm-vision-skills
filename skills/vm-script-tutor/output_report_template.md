## 操作步骤

1. 在 VM 中拖入 **脚本** 模块到流程编辑区，并将其放置在合适位置（前序模块需能提供以下输入变量来源）
2. 双击模块打开配置窗口
3. 在 **输入设置** 中按下表依次添加输入变量：

   | 变量名 | 类型 | 初始值 / 订阅来源 | 说明 |
   | --- | --- | --- | --- |
   | {{in0}} | {{int}} | 订阅前序模块的 xxx 或填默认值 | {{用途}} |
   | ... | ... | ... | ... |

4. 在 **输出设置** 中按下表依次添加输出变量：

   | 变量名 | 类型 | 是否在图像上显示 | 说明 |
   | --- | --- | --- | --- |
   | {{out0}} | {{int}} | 否 | {{用途}} |
   | ... | ... | ... | ... |

5. **（仅当使用第三方库或非默认 BCL 程序集时）** 在已打开的脚本配置窗口中，点击控制栏 → **编辑程序集** → 逐一添加以下程序集：

   | 程序集 / DLL | 来源 / 说明 | 是否已在 csproj 中 |
   | --- | --- | --- |
   | {{OpenCvSharp.dll}} | {{NuGet 包 OpenCvSharp4，需放至 VM 可见路径}} | {{是（无需操作）/ 否（需手动添加）}} |
   | ... | ... | ... |

   - **"是否已在 csproj 中"为"是"**：无需操作，跳过该行
   - **"是否已在 csproj 中"为"否"**：在"编辑程序集"界面点击右上角 **添加** → 找到对应 `.dll` 文件 → 点击打开
   
   > **注意**：若 skill 在生成代码前已提示过"请先添加程序集并重新导出工程"，说明此步骤应在**导出工程前**完成。如当时未操作，请现在补做，完成后在 VM 中重新**导出工程**，然后重新请 skill 替换 `UserScript.cs`，再继续后续步骤。

6. 如交付方式为"直接覆盖了 `UserScript.cs`"：跳过下一步；否则控制栏 → **导入** → 选择生成的脚本文件 `{{文件名}}`
7. 单击 **预编译** → 观察 **编译结果显示** 区域，确认无报错
8. 单击 **执行** 跑一次，确认输出符合预期
9. 单击 **确定** 保存并退出

## 注意事项

- 输入/输出变量名**必须与代码完全一致**（含大小写、数组的 `[]` 标记）。脚本里出现的变量都已在上表中列出
- 如果脚本中 `Process` 在异常分支返回 `false`，模块状态会变红。请用 `try/catch` 包好可能抛错的逻辑
- VS 断点调试：控制栏 → **导出工程** → 用 VS 打开 .sln → 附加到 VM 主进程（详见 `references/csharp_debug.md`）
