# 模块运行参数设置/获取 — 完整工作流

> 本文是 SKILL.md §6 的详细展开，包含步骤 0–4 的完整操作流程、工作流总览图和降级处理汇总。触发条件与强制确认事项以 SKILL.md §6 为准。

## 步骤 0：从映射表查询工具箱和模块英文名（必须执行）

用户通常只知道模块的中文名（如"单点抓取"），而 AlgorithmTab.xml 位于以模块英文名命名的子目录下。因此，**必须先读取 `references/VisionMaster模块映射表.md`**，从中查询模块所属的工具箱和英文名。

**映射表结构**：

| 列名 | 含义 | 示例 |
|------|------|------|
| 工具箱英文名 | AlgorithmTab.xml 路径中的一级子目录 | `Calculation` |
| 工具箱中文名 | 工具箱的显示名称 | 计算 |
| 模块英文名 | AlgorithmTab.xml 路径中的二级子目录 + XML 文件名前缀 | `SinglePointGrabModu` |
| 模块中文名 | 用户在 VM 流程中看到的模块显示名 | 单点抓取 |

**查询流程**：

1. 读取 `references/VisionMaster模块映射表.md`
2. 在表中按用户提到的**模块中文名**查找匹配行
3. 提取该行的**工具箱英文名**、**工具箱中文名**、**模块英文名**
4. **必须显式告知用户**查询结果并请求确认：

   > "已从映射表查询到：
   > - 模块中文名「**{{用户提到的中文名}}**」
   > - 所属工具箱：**{{工具箱英文名}}（{{工具箱中文名}}）**
   > - 模块英文名：**{{模块英文名}}**
   >
   > 请确认以上信息是否准确？"

5. **等待用户确认后**，才能进入步骤 1 定位 AlgorithmTab.xml

**映射表查询失败的降级处理**：

| 情形 | 处理 |
|------|------|
| 模块中文名在映射表中找不到**精确**匹配 | **必须显式告知用户**："映射表中未找到「{{用户提到的中文名}}」的精确匹配，请提供该模块的：①工具箱英文名；②模块英文名。如果模块中文名有多处对应，也请一并说明。" |
| 找到多个匹配（同一个中文名对应多条记录） | 列出所有匹配行（工具箱 + 模块英文名），让用户指认具体是哪一条 |
| 用户提供了模块中文名但映射表中无对应 | 不得自行推测或近似匹配；**必须中断并要求用户提供** |

> **核心原则**：映射表是从中文名到文件路径的唯一桥梁，此步不可跳过。即使用户同时提供了中英文名，仍需通过映射表交叉验证。

---

## 步骤 1：定位并读取 AlgorithmTab.xml 文件

根据步骤 0 确认的**工具箱英文名**和**模块英文名**，构造 AlgorithmTab.xml 的完整路径：

```
{VM根目录}\Applications\Module(sp)\x64\{工具箱英文名}\{模块英文名}\{模块英文名}AlgorithmTab.xml
```

**示例**：用户要设置"单点抓取"模块的参数：
- 步骤 0 确认：工具箱 = `Calculation`，模块英文名 = `SinglePointGrabModu`
- 构造路径：`C:\Program Files\VisionMaster4.3.0\Applications\Module(sp)\x64\Calculation\SinglePointGrabModu\SinglePointGrabModuAlgorithmTab.xml`

**路径校验规则**：

1. 按 §1 确认的 VM 版本，确定安装根目录（如 `C:\Program Files\VisionMaster4.3.0`）
2. 检查 `{VM根目录}\Applications\Module(sp)\x64\{工具箱英文名}\` 目录是否存在
   - **若工具箱目录不存在**：**必须显式告知用户**："在 `{VM根目录}\Applications\Module(sp)\x64\` 下未找到工具箱目录 `{工具箱英文名}`，请确认工具箱英文名是否正确。当前从映射表查询到的工具箱英文名为 `{工具箱英文名}`，如需修正请重新提供。"
3. 检查 `{模块英文名}\` 子目录是否存在
   - **若模块目录不存在**：**必须显式告知用户**："在工具箱 `{工具箱英文名}` 下未找到模块目录 `{模块英文名}`，请确认模块英文名是否正确。当前从映射表查询到的模块英文名为 `{模块英文名}`，如需修正请重新提供。"
4. 检查 `{模块英文名}AlgorithmTab.xml` 文件是否存在
   - **若 XML 文件不存在**：**必须显式告知用户**："在模块目录 `{模块英文名}` 下未找到 `{模块英文名}AlgorithmTab.xml`，请确认文件名是否正确，或直接提供 XML 文件内容。"
5. 读取该 XML 文件（**允许**，这是本 skill 的授权操作）

> 备用方案：若映射表查询成功但路径校验失败（目录名可能因 VM 版本不同而有差异），可在 `{VM根目录}\Applications\Module(sp)\x64\` 下用 Glob 递归搜索 `*{模块英文名}AlgorithmTab.xml` 进行兜底。

## 步骤 2：从 XML 中提取参数信息

AlgorithmTab.xml 的典型节点结构示例：

```xml
<Param>
  <Name>AngleStart</Name>        <!-- 参数 Key，SetValue 第一个参数 -->
  <DisplayName>起始角度</DisplayName>
  <MinValue>-180</MinValue>       <!-- 允许的最小值 -->
  <MaxValue>180</MaxValue>        <!-- 允许的最大值 -->
  <DefaultValue>-30</DefaultValue>
  <ParamType>Float</ParamType>
</Param>
```

提取规则：
- **参数 Key = `<Name>` 节点内容**（大小写敏感，原样使用）
- **取值范围 = `<MinValue>` 到 `<MaxValue>`**
- 若 XML 中不含 MinValue/MaxValue，说明无范围约束，可直接使用用户输入值

---

## 步骤 3：范围校验

用户提供的参数值**超出 MinValue–MaxValue 范围时**：
- **必须明确告知用户**：参数值超出范围，正确范围是 X 到 Y
- 不得生成超限代码；等用户修正后再生成

---

## 步骤 4：生成代码

确认参数名（Key）和合法值后，生成如下代码（所有参数值均以字符串传入）：

```csharp
// 示例：设置单点抓取模块的示教点坐标
CurrentProcess.GetModule("单点抓取1").SetValue("TeachPointX", "100");
CurrentProcess.GetModule("单点抓取1").SetValue("TeachPointY", "100");
CurrentProcess.GetModule("单点抓取1").SetValue("TeachPointZ", "1");
```

## 完整工作流总览

```
用户提到模块中文名（如"单点抓取"）
    │
    ▼
（向用户确认：模块显示名称 + 参数描述 + 目标值）
    │
    ▼
步骤 0：查映射表 → 工具箱 + 模块英文名 → 显式告知用户确认
    │ （用户确认）
    ▼
步骤 1：构造路径 → 校验目录和文件存在 → 读取 AlgorithmTab.xml
    │ （目录/文件不存在则降级处理，问用户）
    ▼
步骤 2：从 XML 提取参数 Key（<Name>）和范围（MinValue/MaxValue）
    │
    ▼
步骤 3：校验用户输入值是否在范围内（超限则阻断，等用户修正）
    │
    ▼
步骤 4：生成 SetValue/GetValue 代码
```

## 降级处理汇总

| 阶段 | 情形 | 处理 |
|------|------|------|
| 步骤 0 | 模块中文名在映射表中找不到精确匹配 | **显式告知用户**："映射表中未找到「xxx」的精确匹配，可能原因：①用户使用的别名与表中中文名不一致；②该模块为 VM 4.4 / 5.0 等后续版本新增的模块（本表基于 VM 4.3）。请提供该模块的：①工具箱英文名；②模块英文名" |
| 步骤 0 | 找到多个同名匹配 | 列出所有匹配行，让用户指认 |
| 步骤 1 | 工具箱目录不存在 | **显式告知用户**："未找到工具箱目录 `{工具箱英文名}`，请确认工具箱英文名是否正确" |
| 步骤 1 | 模块目录不存在 | **显式告知用户**："在工具箱 `{工具箱英文名}` 下未找到模块目录 `{模块英文名}`，请确认模块英文名是否正确" |
| 步骤 1 | AlgorithmTab.xml 文件不存在 | **显式告知用户**："在该目录下未找到 AlgorithmTab.xml，请确认文件名或直接提供 XML 文件内容" |
| 步骤 2 | XML 中找不到目标参数节点 | **显式告知用户**："在 XML 中未找到对应参数，请提供：①参数的英文名称（Key）；②参数值" |
| 步骤 2 | XML 中找不到 MinValue / MaxValue | 无范围约束，使用用户输入值，并在代码注释中注明"参数范围未知，值来自用户输入" |
| 步骤 3 | 用户输入值超出 MinValue–MaxValue | **显式告知用户**：参数值超出范围，正确范围是 X 到 Y；等用户修正 |
| 任意阶段 | 用户明确提供了参数 Key（跳过查询） | 仍尝试查阅 XML 确认范围；若用户坚持跳过，记录"参数名由用户提供，未经 XML 验证" |

> **核心原则**：任何一步失败，都要**显式告知用户缺失了什么**，并明确说明需要提供的信息（工具箱英文名 / 模块英文名 / 参数英文名 / 参数值），不得自行推断、跳过或用占位符代替。
