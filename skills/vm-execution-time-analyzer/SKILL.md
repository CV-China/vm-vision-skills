# VisionMaster 方案耗时分析

## 概述

本 skill 对 VisionMaster 方案的运行耗时进行自动化分析，输出固定格式的 HTML 交互式报表。

### 输入要求

| 输入 | 说明 | 必填 |
|------|------|------|
| `ModuleFrame*.log` | 算法耗时日志目录（通常在 `\VisionMaster4.4.0\Applications\log\ModuleProxy`） | ✅ |

> **注意**：触发 skill 后，如果用户未提供路径，应主动询问日志目录路径。如果项目目录下已有文件，可将其作为快捷选项提示用户。

### 输出

一份**自包含的 HTML 报告**，通过 CDN 加载 ECharts，包含两大板块：
1. 整体方案耗时评估（流程级/模块级汇总 + 自动评估结论）
2. 耗时分析图谱（可交互的柱状图/饼图/多维度对比图，支持模块筛选）

## 使用方法

### 交互流程（触发 skill 时优先使用）

当用户触发此 skill 但未提供明确的文件路径时，**必须依次询问**以下问题，获取用户输入后再执行分析：

> **1. 请提供日志文件路径：**
> （包含 `ModuleFrame*.log` 的目录，通常在 VisionMaster 安装目录下：
> `\VisionMaster4.4.0\Applications\log\ModuleProxy`）

> **2. 请选择分析日志时间段：**（现场日志可能非常大，选择合理的时间段可加快分析速度）
>
> - ① 近一小时
> - ② 近3小时
> - ③ 近一天
> - ④ 近一周
> - ⑤ 所有时间段

用户提供路径后，自动代入以下命令执行（输出文件名自动按 `耗时分析报告_<时间>.html` 命名）：

```bash
python scripts/analyze.py \
  --log <用户提供的日志目录> \
  --time-range <用户选择的时间段>
```

时间段映射：① → `1h`，② → `3h`，③ → `1d`，④ → `1w`，⑤ → `all`（默认）。

如果用户一次性提供了所有信息，直接执行，无需重复询问。

### 直接运行分析脚本（已知路径时）

```bash
python scripts/analyze.py --log 输入信息/ModuleProxy/
```
输出文件默认生成到桌面 `耗时分析输出/耗时分析报告_<时间>.html`。

### 控制台输出

> **⚠️ Git Bash 编码前置**：Python print 中文在 Git Bash 下可能显示为乱码。
> 运行脚本前必须先设置编码：
> ```bash
> export PYTHONIOENCODING=utf-8
> ```
> 或使用 PowerShell：`$env:PYTHONIOENCODING = "utf-8"`

运行脚本后，**先生成纯文本评估结论打印到控制台**，然后再渲染 HTML 报告。用户无需打开浏览器即可快速查看核心结论：

```

  [整体评估结论]
========================================================

1. 耗时波动影响最大的流程
流程10001（高影响），平均耗时 66.711ms（共8个模块，执行14830次）

2. 流程内波动影响最大的模块
模块ID:14 「高精度匹配」（高影响），平均耗时 52.051ms

3. 最慢前五次中波动拖累最大的模块
模块ID:14 「高精度匹配」，最慢一次耗时 203.613ms（平均耗时 151.6ms）

--------------------------------------------------------
  总模块数: 15  |  总流程数: 2  |  总执行次数: 14830

```

### 参数说明

| 参数 | 必填 | 说明 | 默认值 |
|------|------|------|--------|
| `--log` | ✅ | ModuleFrame*.log 所在目录 | - |
| `--output` | ❌ | 输出 HTML 路径 | `桌面/耗时分析输出/耗时分析报告_<时间>.html` |
| `--time-range` | ❌ | 分析时间段：`1h`/`3h`/`1d`/`1w`/`all` | `all` |
| `--exclude-n` | ❌ | 排除前 N 次 VM 预热运行 | `5` |

## 输入文件准备

在项目 `输入信息/` 目录下准备好以下文件：

```
输入信息/
└── ModuleProxy/
    ├── ModuleFrame.log     # 算法耗时日志
    ├── ModuleFrame.1.log   # (可选) 滚动日志
    ├── ModuleFrame.2.log   # (可选) 滚动日志
    └── ModuleProxy.log     # (可选) 仅用于参考
```

### 何时生成新日志

当需要最新耗时数据时，按以下步骤生成：

1. 在 VisionMaster 中打开目标方案
2. 连续运行方案一段时间（建议 > 100 次以获得稳定的统计数据）
3. 在 VM 安装目录的 `Log/ModuleProxy/` 下找到 `ModuleFrame*.log` 文件
4. 将日志文件复制到 `输入信息/ModuleProxy/` 目录

## 报告功能详解

### 板块一：整体方案耗时评估

- **术语解释面板**：报告顶部提供可展开的 📖 名词解释，通俗说明 CV、P95、标准差、极差等指标含义，面向非技术用户
- **评估结论**：基于耗时占比和波动性自动生成的自然语言评估（避免抽象术语，用大白话描述结论）
- **流程级汇总表**：每个流程的模块数、执行次数、Avg/Max/Min、占比、高影响模块（表头悬停显示指标解释）
- **模块级明细表**：每个模块的完整统计指标（含流程占比），支持按流程筛选

### 板块二：耗时分析图谱

基于 **ECharts** 的交互式图表：

| 图表 | 类型 | 内容 |
|------|------|------|
| 流程耗时分布 | 柱状图 + 饼图 | 各流程 Avg/Max/P95 对比 + 流程耗时占比 |
| 模块耗时分析 | 柱状图 + 饼图 | 各模块 Max/Avg/Min 对比 + 按模块类型耗时占比 |
| 耗时曲线 | 折线图 | 模块/流程耗时时间序列，支持 dataZoom 缩放 |

**交互功能**：
- 流程和模块筛选下拉框，支持多选
- 点击模块标签筛选特定模块的图表显示
- 悬停显示详细 tooltip
- 浏览器窗口大小变化时图表自动适配

### 板块零：图例与名词解释

报告页面顶部包含：
- **📖 展开名词解释**：可展开/收起的术语面板，用生活化比喻解释每个指标的含义
- **波动等级图例栏**：直观展示 🟢🟡🔴 三级波动的阈值和含义

## 关键指标通俗解释

| 指标 | 通俗解释 | 如何使用 |
|------|---------|---------|
| **平均耗时 (Avg)** | "这个模块通常跑多快" | 越低越好，是性能优化的基准线 |
| **变异系数 (CV)** | "这个模块每次跑得稳不稳" | < 20% 很稳，> 50% 需要排查 |
| **P95** | "95%的情况下最快能跑多快" | 比最大值更能反映真实体验 |
| **标准差 (Std)** | "每次跑的波动有多大" | 值越大说明忽快忽慢 |
| **极差** | "最慢和最快的差距" | 差距大 = 有偶发慢速 |
| **超2倍均值次数** | "跑得特别慢的次数" | 越多说明偶发卡顿越频繁 |
| **耗时占比** | "这个流程/模块占用了多少总时间" | 占比越高越是优化重点 |
| **流程占比** | "该模块占所在流程总耗时的比例" | 占比大的模块如果波动高，对整体稳定性影响就大 |
| **波动贡献指数 (VCI)** | "这个模块对整体稳定性的实际影响有多大" | VCI = CV × 流程占比，综合考量波动程度和重要性 |

## 波动评级标准

### 自身波动评级（基于 CV）

| 变异系数 (CV = Std/Avg) | 评级 | 含义 |
|-------------------------|------|------|
| < 20% | 🟢 低波动 | 模块自身耗时稳定 |
| 20% ~ 50% | 🟡 中波动 | 存在一定波动，建议关注 |
| > 50% | 🔴 高波动 | 波动较大，需排查偶发性异常 |

### 波动影响评级（基于 VCI = CV × 流程占比，流程内百分位排序）

> **核心理念**：单纯看 CV 不够——一个 CV 很高但耗时占比极小的模块，对整体方案稳定性的实际影响有限。VCI 综合了"波动程度"和"耗时占比"两个维度，帮你找到真正导致整体方案忽快忽慢的元凶。
>
> **为什么用百分位排序而非固定阈值？** 不同流程的模块数量差异很大（有的流程只有2-3个模块，有的流程有几十个）。固定阈值（如 VCI>15%=高影响）在模块少的流程中无人达标、在模块多的流程中可能大面积飘红。改用流程内排序后，无论模块多少，每个流程都能公平地识别出影响最大的那几个。

| VCI 排序位置 | 评级 | 含义 |
|-------------|------|------|
| 流程内前 5% | 🔴 高影响 | 该流程内对稳定性影响最大的模块，需优先排查优化 |
| 流程内 5% ~ 20% | 🟡 中影响 | 对稳定性有一定影响，建议列入观察列表 |
| 流程内后 80% | 🟢 低影响 | 对整体稳定性影响较小 |

> **流程级波动影响**：每个流程也有自己的波动影响评级。流程 VCI = 流程自身的 CV × 该流程耗时占全部方案耗时的比例，跨所有流程按同样的百分位排序规则划分影响等级。这样既能看模块维度，也能从流程维度评估哪里最"痛"。

## 技术架构

```
ModuleFrame*.log
     │
     v
log_parser.py
(正则匹配模块行/流程行)
     │
     v
原始时间序列数据
     │
     v
time_analyzer.py
(百分位/波动/CV/评估)
     │
     v
report_generator.py
(ECharts → HTML)
```

### 依赖关系

| 依赖 | 来源 | 用途 |
|------|------|------|
| ECharts 5.x | CDN | 交互式图表渲染 |
| Python 3.7+ | 系统 | 日志解析 + 报告生成 |

## 核心脚本说明

| 文件 | 职责 |
|------|------|
| `scripts/analyze.py` | 主入口，CLI 参数处理 |
| `scripts/log_parser.py` | ModuleFrame 日志解析引擎，对标 LogTimeAnalyzer C# 逻辑 |
| `scripts/time_analyzer.py` | 统计计算（百分位/CV/波动评级）+ 评估结论生成 |
| `scripts/report_generator.py` | HTML 报告组装（图表数据/表格） |
| `templates/report_template.html` | HTML 模板（ECharts 集成） |
| `references/module_names.json` | VM 模块类名→中文名映射表 |

## 注意事项

1. **CDN 依赖**：HTML 报告需联网加载 ECharts（首次加载后浏览器会缓存）
> **⚠️ 离线环境**：HTML 报告需联网加载 ECharts（首次后浏览器缓存）。
> 离线时图表区为空，但数据表格和评估结论仍可正常查看。
> 如须离线使用，可在生成后手动替换 CDN 链接为本地 ECharts 文件。
2. **日志格式兼容**：当前支持 VisionMaster 4.4.0+ 的 ModuleFrame 日志格式
3. **数据精度**：统计指标保留 3 位小数，对标 LogTimeAnalyzer 的 `Math.Round(x, 3)`
4. **内存使用**：大型日志文件（> 100MB）建议使用合理的 `--exclude-n` 值
