# VM Sol 文件解析与方案理解 Skill 集合

以已有的 VM 4.3.0 sol 解析对比工具为基础，结合 AI 对 VM 4.4.0 方案的对比分析和扩展，开发了两个互补的 Skill：

### vm-sol-format — 方案文件操作

- **方案解析**：.sol → 结构化输出，提取模块拓扑、参数值、连线、订阅、脚本、流程配置等全部信息
- **方案对比**：两个方案 diff，定位模块增删、参数变更、连线变化
- **方案修改**：修改参数/订阅/绑定/模块生命周期后回写 sol，支持加密

### vm-sol-structure — 方案知识库

- VM 模块索引（类名-中文名对照、帮助文档关联）
- 模块连线机制、参数类型体系等专题知识
- 让 AI 理解每个模块"做什么"，而不只是"有什么参数"

### 已验证可行的方向（基于旧版本 Skill，由同事修改或扩展实现，部分内容已吸收到当前 Skill）

| --------------------- | -------------------------------- | ---------- |
| 方案对比              | 快速定位版本间差异               | Luowei     |

### 当前 Skill 能做什么

```
CLI 4 个命令：parse / compare / inspect / modify
modify 8 个操作：参数值、脚本、订阅、绑定、增模块、改名称、启禁用、删模块
全部通过 VM 4.4.0 实际验证，加密/非加密均支持
```

### 后续可扩展方向

| 方向           | 说明                                                    |
| -------------- | ------------------------------------------------------- |
| 母版数据方案   | VM 接口生成全模块默认参数库，CLI 内置，任意模块零模板新建 |
| 动态 IO 增删   | 模块输入/输出端口动态增删                               |
| Group/流程操作 | 扩展增删拷到所有模块类型                                |
| CLI 工具生成   | 将常用操作封装为独立 CLI 工具，方便批量处理和自动化集成 |
| 方案理解与梳理 | AI 读懂方案逻辑，输出流程说明，给出优化建议             |
| 模块知识查询   | 某模块有什么参数、干什么用，直接问 AI                   |
| 方案自动生成   | 描述需求，直接生成 sol 方案（远期）                     |

**核心思路：** 让 AI 能"看懂" sol 方案，做到人通过 VM 软件能了解的信息，AI 通过解析 sol 同样能了解。
